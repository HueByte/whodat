# WhodatDbModelSnapshot

> **File:** `src/api/Whodat.Api/Data/Migrations/WhodatDbModelSnapshot.cs`

This generated class stores a snapshot of the WhodatDbContext model as of a migration, enabling EF Core to detect and script changes when future migrations are created. It enumerates the entities, keys, indexes, and provider-specific column types (including ASP.NET Identity tables) that comprise the current model.

## Remarks

The BuildModel method reconstructs the entire model graph inside a ModelBuilder, including identity-related entities such as IdentityRole, `IdentityRoleClaim<string>`, `IdentityUserClaim<string>`, `IdentityUserLogin<string>`, `IdentityUserRole<string>`, and their mappings to the AspNetRoles, AspNetRoleClaims, AspNetUserClaims, AspNetUserLogins, and AspNetUserRoles tables. Annotations like ProductVersion reflect the EF Core tooling version used at scaffolding time. The column types (TEXT and INTEGER) indicate the SQLite provider in use for this snapshot.

Snapshots are auto-generated and should not be manually edited. They are used by migrations to compare the current model against the last-known snapshot to produce the necessary migration operations. Renaming the DbContext or altering the model without creating a corresponding migration will desynchronize the snapshot and migrations.

Note that the file is partial and tied to WhodatDbModelSnapshot, so changes are typically made via model changes and Add-Migration rather than editing directly.
